วิธีใช้งาน LINE Drive Bot

ระบบนี้ใช้สำหรับเก็บไฟล์ที่สมาชิกส่งในกลุ่ม LINE ลง Google Drive อัตโนมัติ โดยแยกเป็นโฟลเดอร์ Images, Videos, และ Documents

1. เตรียม Google Drive

  1. เข้า Google Cloud Console
  2. สร้าง Project ใหม่ หรือใช้ Project เดิม
  3. เปิดใช้งาน Google Drive API
  4. สร้าง Service Account
  5. สร้าง key แบบ JSON แล้วดาวน์โหลดไฟล์เก็บไว้
  6. เปิด Google Drive แล้วสร้างโฟลเดอร์หลัก เช่น LINE Group Files
  7. แชร์โฟลเดอร์หลักให้ email ของ Service Account เป็น Editor
  8. คัดลอก Folder ID จาก URL ของโฟลเดอร์

ตัวอย่าง URL ของโฟลเดอร์ Google Drive:

https://drive.google.com/drive/folders/1AbCdEfGh12345

ค่า 1AbCdEfGh12345 คือค่าที่ต้องนำไปใส่ใน GOOGLE_DRIVE_ROOT_FOLDER_ID

2. เตรียม LINE Bot

  1. เข้า LINE Developers Console
  2. สร้าง Provider และ Messaging API Channel
  3. คัดลอกค่า Channel secret
  4. สร้างหรือคัดลอก Channel access token
  5. เปิดใช้งาน webhook
  6. ปิด Auto-reply messages ถ้าไม่ต้องการให้บอตตอบข้อความเอง
  7. เพิ่มบอตเข้าไปในกลุ่ม LINE ที่ต้องการเก็บไฟล์

3. ตั้งค่าไฟล์ .env

คัดลอกไฟล์ตัวอย่าง:

cp .env.example .env

แก้ไขไฟล์ .env ให้มีค่าประมาณนี้:

PORT=3000

LINE_CHANNEL_SECRET=ใส่_channel_secret
LINE_CHANNEL_ACCESS_TOKEN=ใส่_channel_access_token

GOOGLE_DRIVE_ROOT_FOLDER_ID=ใส่_google_drive_folder_id

GOOGLE_SERVICE_ACCOUNT_BASE64=ใส่_service_account_json_แบบ_base64

แนะนำให้ใช้ credential แบบ base64 เพราะใส่ในไฟล์ .env ง่ายกว่า JSON หลายบรรทัด:

base64 -w 0 service-account.json

จากนั้นนำผลลัพธ์ที่ได้ไปใส่ใน GOOGLE_SERVICE_ACCOUNT_BASE64

4. ติดตั้งและรันระบบ

ติดตั้ง dependencies:

npm install

รันระบบ:

npm start

ถ้ารันสำเร็จ จะเห็นข้อความประมาณนี้:

LINE Drive bot listening on port 3000

5. เปิดให้ LINE เรียก webhook ได้

LINE ต้องเรียก webhook ผ่าน URL ที่เข้าถึงจากอินเทอร์เน็ตได้และเป็น HTTPS ถ้าทดสอบบนเครื่องตัวเอง ให้ใช้ ngrok หรือ cloudflared

ตัวอย่างใช้ ngrok:

ngrok http 3000

นำ URL ที่ได้ไปตั้งใน LINE Developers โดยเติม /webhook ต่อท้าย:

https://xxxx.ngrok-free.app/webhook

6. ทดสอบใช้งาน

  1. เพิ่มบอตเข้าไปในกลุ่ม LINE
  2. ส่งรูปภาพในกลุ่ม แล้วตรวจโฟลเดอร์ Images
  3. ส่งวิดีโอในกลุ่ม แล้วตรวจโฟลเดอร์ Videos
  4. ส่งไฟล์ PDF, Word, Excel หรือไฟล์อื่น แล้วตรวจโฟลเดอร์ Documents

7. โครงสร้างไฟล์บน Google Drive

LINE Group Files/
  Images/
  Videos/
  Documents/

8. คำสั่งจัดระเบียบไฟล์

ส่งคำสั่งในกลุ่ม LINE เพื่อกำหนดหมวดสำหรับไฟล์ที่จะส่งต่อจากนั้น:

/folder งานประชุม

หลังจากตั้งหมวดแล้ว ไฟล์ถัดไปจะเข้าโฟลเดอร์ย่อยตามประเภทไฟล์:

LINE Group Files/
  Images/
    งานประชุม/
  Videos/
    งานประชุม/
  Documents/
    งานประชุม/

คำสั่งที่ใช้ได้:

/folder       ดูหมวดปัจจุบัน
/folder off   ปิดหมวดปัจจุบัน
/help         ดูคำสั่งทั้งหมด

ถ้าไม่ใช้คำสั่ง /folder ไฟล์จะถูกเก็บในโฟลเดอร์ปกติตามประเภทไฟล์ ถ้าตั้งหมวดที่ยังไม่มีโฟลเดอร์อยู่ ระบบจะสร้างโฟลเดอร์ให้อัตโนมัติ เมื่อใช้งานหมวดเสร็จ ให้พิมพ์ /folder off

หลังอัปโหลดสำเร็จ บอตจะรอช่วงสั้น ๆ แล้วสรุปผลเป็นข้อความเดียวพร้อมลิงก์โฟลเดอร์ Google Drive:

ทำการบันทึกไฟล์เรียบร้อย 10 ไฟล์

สรุป:
Images/งานประชุม: 6 ไฟล์
Videos/งานประชุม: 2 ไฟล์
Documents/งานประชุม: 2 ไฟล์

โฟลเดอร์:
Images/งานประชุม: https://drive.google.com/drive/folders/...
Videos/งานประชุม: https://drive.google.com/drive/folders/...
Documents/งานประชุม: https://drive.google.com/drive/folders/...

ค่าเริ่มต้นจะรอ 45 วินาทีก่อนส่งสรุป ปรับได้ด้วยค่า SUMMARY_DELAY_SECONDS ถ้าบันทึกไฟล์ไม่สำเร็จ บอตจะแจ้ง error ในกลุ่ม

ถ้าต้องการให้สมาชิกในกลุ่มเปิดลิงก์ได้ ต้องแชร์โฟลเดอร์ Google Drive ให้สมาชิกเหล่านั้น หรือปรับสิทธิ์เป็น anyone with the link ตามนโยบายความปลอดภัยของคุณ

9. คำสั่งนัดหมายในกลุ่ม

คำสั่งนัดหมายต้องตั้งค่า MySQL ก่อนใช้งาน เพื่อให้ข้อมูลนัดหมายไม่หายหลัง server restart

รูปแบบเวลาที่ใช้ในคำสั่งคือ YYYY-MM-DD HH:mm และอ้างอิงเวลาไทย Asia/Bangkok

สร้างนัดหมาย:

/นัด ประชุมโครงการ | 2026-07-05 14:00 | ห้องประชุม A

ดูนัดหมายที่กำลังจะมาถึง:

/นัดหมาย

ยกเลิกนัดหมาย:

/ยกเลิกนัด A-1234

บอตจะแจ้งเตือนในกลุ่ม 2 ครั้ง:

เช้าวันนัดหมาย เวลา 08:00
ก่อนเวลานัดหมาย 10 นาที

ตัวแปรที่เกี่ยวข้อง:

DB_HOST=your_mysql_host
DB_PORT=3306
DB_USER=your_mysql_user
DB_PASSWORD=your_mysql_password
DB_NAME=your_mysql_database

REMINDER_MORNING_HOUR=8
REMINDER_CHECK_INTERVAL_SECONDS=60

วิธีทดสอบหลัง deploy:

  1. พิมพ์ /นัด help ในกลุ่ม LINE
  2. สร้างนัดหมายทดสอบที่อยู่ในอนาคต เช่น อีก 15 นาที
  3. พิมพ์ /นัดหมาย เพื่อตรวจว่าบอตบันทึกนัดหมายแล้ว
  4. รอให้บอตแจ้งเตือนก่อนเวลานัดหมาย 10 นาที
  5. ทดสอบยกเลิกด้วย /ยกเลิกนัด รหัส

ถ้ายังไม่ได้ตั้งค่า MySQL บอตยังเก็บไฟล์ได้ตามปกติ แต่คำสั่งนัดหมายจะยังใช้งานไม่ได้ การแจ้งเตือนใช้ LINE push message จึงนับรวมกับ quota/แพ็กเกจข้อความของ LINE Official Account

10. ปัญหาที่พบบ่อย

Webhook Verify ไม่ผ่าน: ตรวจว่า URL ลงท้ายด้วย /webhook และ server รันอยู่

ขึ้น Invalid LINE signature: ตรวจค่า LINE_CHANNEL_SECRET ว่าตรงกับ channel ที่ใช้จริง

อัปโหลดเข้า Drive ไม่ได้: ตรวจว่าแชร์โฟลเดอร์หลักให้ email ของ Service Account เป็น Editor แล้ว

ไม่เห็นไฟล์ใน Drive: ตรวจ log ของ server และตรวจว่าไฟล์ที่ส่งเป็นรูป วิดีโอ หรือไฟล์แนบจริง

คำสั่งนัดหมายบอกว่ายังไม่ได้เปิดใช้งาน: ตรวจค่า DB_HOST, DB_USER, DB_PASSWORD, และ DB_NAME บน Hostinger

แจ้งเตือนไม่เข้า: ตรวจ log ของ server, ตรวจว่า app ยัง Running และตรวจว่า LINE push message ยังไม่เกิน quota

หลัง deploy จริง แนะนำให้ใช้ URL ถาวร เช่น Render, Railway, Fly.io, VPS หรือ platform อื่นที่รองรับ HTTPS แล้วนำ URL ถาวรนั้นไปตั้งเป็น webhook ใน LINE Developers
