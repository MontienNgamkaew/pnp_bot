คู่มือติดตั้ง PNP BOT บน Hostinger

คู่มือนี้อธิบายตั้งแต่การเตรียม Google Drive, การตั้งค่า LINE Messaging API, การนำโค้ดขึ้น GitHub, การ deploy บน Hostinger และการทดสอบใช้งานจริง

คู่มือนี้อ้างอิงโค้ดในโปรเจกต์นี้ ซึ่งรันด้วย Node.js และมี endpoint หลักคือ /webhook

ภาพรวมระบบ

สมาชิกส่งไฟล์ในกลุ่ม LINE
        ↓
LINE ส่ง webhook มาที่ Hostinger
        ↓
Node.js bot ตรวจลายเซ็นและดาวน์โหลดไฟล์จาก LINE
        ↓
bot อัปโหลดไฟล์เข้า Google Drive
        ↓
Drive แยกโฟลเดอร์ Images, Videos, Documents

สิ่งที่ต้องมี

  • บัญชี Hostinger ที่รองรับ Node.js app และเชื่อม GitHub ได้
  • โดเมนหรือซับโดเมน เช่น bot.example.com
  • บัญชี GitHub
  • บัญชี LINE Developers
  • บัญชี Google Cloud และ Google Drive
  • Node.js 20 ขึ้นไปสำหรับทดสอบบนเครื่องตัวเอง

2. ตั้งค่า LINE Messaging API

2.1 สร้าง Provider และ Channel

  1. เข้า https://developers.line.biz/console/
  2. สร้าง Provider ใหม่ หรือใช้ Provider เดิม
  3. กด Create a Messaging API channel
  4. กรอกชื่อบอต รายละเอียด และข้อมูลที่ LINE ต้องการ
  5. สร้าง channel ให้เรียบร้อย

2.2 คัดลอก Channel Secret

  1. เข้า channel ที่สร้างไว้
  2. ไปที่แท็บ Basic settings
  3. คัดลอกค่า Channel secret
  4. ค่านี้ใช้เป็น LINE_CHANNEL_SECRET

2.3 สร้าง Channel Access Token

  1. เข้าแท็บ Messaging API
  2. หา section Channel access token
  3. กด Issue หรือ Reissue
  4. คัดลอก token ที่ได้
  5. ค่านี้ใช้เป็น LINE_CHANNEL_ACCESS_TOKEN

2.4 เปิด Webhook

  1. ในแท็บ Messaging API
  2. ตั้ง Use webhook เป็น Enabled
  3. ปิด Auto-reply messages ถ้าไม่ต้องการให้บอตตอบข้อความเอง
  4. ปิด Greeting messages ถ้าไม่ต้องการข้อความทักทายอัตโนมัติ

2.5 เพิ่มบอตเข้ากลุ่ม LINE

  1. ในแท็บ Messaging API หา QR code ของบอต
  2. เพิ่มบอตเป็นเพื่อน
  3. เชิญบอตเข้ากลุ่มที่ต้องการเก็บไฟล์

5. Deploy บน Hostinger ผ่าน GitHub

ชื่อเมนูใน hPanel อาจต่างกันเล็กน้อยตามแพ็กเกจ Hostinger แต่ค่าที่ต้องตั้งจะเหมือนกัน: repository, branch, install command, start command, environment variables และ domain

5.0 สร้าง MySQL Database สำหรับระบบนัดหมาย

  1. เข้า Hostinger hPanel
  2. ไปที่เมนู Databases หรือ MySQL Databases
  3. สร้าง database ใหม่ เช่น line_bot
  4. สร้าง user และ password สำหรับ database นี้
  5. จดค่า host, port, database name, username และ password
  6. นำค่าเหล่านี้ไปใส่ใน Environment Variables ของ Node.js app

บอตจะสร้างตาราง appointments ให้อัตโนมัติเมื่อ app start ครั้งแรก

ถ้ายังไม่ต้องการใช้ระบบนัดหมาย สามารถเว้นค่า MySQL ไว้ก่อนได้ บอตจะยังเก็บไฟล์เข้า Google Drive ได้ตามปกติ แต่คำสั่ง /นัด, /นัดหมาย และ /ยกเลิกนัด จะยังไม่ทำงานจนกว่าจะตั้งค่า DB ครบ

5.1 สร้าง Node.js App

  1. เข้า Hostinger hPanel
  2. เลือกเว็บไซต์หรือ hosting plan ที่ต้องการใช้
  3. ไปที่เมนูสำหรับ Node.js, Web App, App Deployment หรือ GitHub Deployment
  4. เลือกสร้าง app ใหม่
  5. เลือก runtime เป็น Node.js 20 หรือสูงกว่า ถ้ามีให้เลือก
  6. เลือก deploy จาก GitHub
  7. เชื่อมต่อบัญชี GitHub และอนุญาตให้ Hostinger อ่าน repository
  8. เลือก repository ของบอต
  9. เลือก branch เป็น main

5.2 ตั้งค่า build และ start command

รายการ ค่า
Install command npm install
Build command เว้นว่าง หรือใส่ npm install ถ้า Hostinger บังคับให้กรอก
Start command npm start
Entry point src/server.js
Port ถ้า Hostinger กำหนด port ให้ ใช้ตัวแปร PORT ตามระบบได้เลย

5.3 ตั้ง Environment Variables บน Hostinger

เพิ่มค่าต่อไปนี้ในหน้า Environment Variables ของ app:

Variable คำอธิบาย
LINE_CHANNEL_SECRET Channel secret จาก LINE Developers
LINE_CHANNEL_ACCESS_TOKEN Channel access token จาก LINE Developers
GOOGLE_DRIVE_ROOT_FOLDER_ID Folder ID ของโฟลเดอร์หลักใน Google Drive
GOOGLE_SERVICE_ACCOUNT_BASE64 JSON key ของ service account ที่แปลงเป็น base64
DB_HOST MySQL host จาก Hostinger สำหรับระบบนัดหมาย
DB_PORT ปกติใช้ 3306
DB_USER MySQL username
DB_PASSWORD MySQL password
DB_NAME ชื่อ MySQL database
REMINDER_MORNING_HOUR ชั่วโมงแจ้งเตือนเช้าวันนัดหมาย เช่น 8
REMINDER_CHECK_INTERVAL_SECONDS ความถี่ที่ server ตรวจนัดหมาย เช่น 60

ถ้า Hostinger ให้ตั้ง PORT เอง ให้ใส่:

PORT=3000

ถ้า Hostinger จัดการ port เอง ไม่ต้องใส่ PORT

ตัวอย่าง environment variables สำหรับระบบนัดหมาย:

DB_HOST=your_hostinger_mysql_host
DB_PORT=3306
DB_USER=your_mysql_user
DB_PASSWORD=your_mysql_password
DB_NAME=your_mysql_database

REMINDER_MORNING_HOUR=8
REMINDER_CHECK_INTERVAL_SECONDS=60
หลังแก้ Environment Variables บน Hostinger ให้ redeploy หรือ restart app ทุกครั้ง ถ้า DB ตั้งค่าถูกต้อง log ตอนเริ่มระบบควรมีข้อความ Appointment database is ready

5.4 ตั้ง Domain หรือ Subdomain

  1. สร้าง subdomain เช่น bot.yourdomain.com
  2. ผูก subdomain นั้นกับ Node.js app
  3. เปิด SSL/HTTPS ใน Hostinger
  4. รอ DNS และ SSL ทำงานสมบูรณ์

5.5 Deploy

  1. กด Deploy หรือ Save แล้วให้ Hostinger เริ่ม deploy
  2. รอจนสถานะเป็น Running หรือ Active
  3. เปิด URL หลักของ app ใน browser

ถ้าระบบทำงาน ควรเห็น JSON ประมาณนี้:

{
  "ok": true,
  "service": "PNP BOT"
}

8. คำสั่งจัดระเบียบไฟล์ในกลุ่ม LINE

ถ้าต้องการแยกไฟล์ตามงานหรือหัวข้อ ให้พิมพ์คำสั่งในกลุ่มก่อนส่งไฟล์: 

/folder งานประชุม

จากนั้นไฟล์ที่ส่งถัดไปจะถูกเก็บเป็นโฟลเดอร์ย่อย:

LINE Group Files/
  Images/
    งานประชุม/
  Videos/
    งานประชุม/
  Documents/
    งานประชุม/

คำสั่งที่ใช้ได้:

/folder       ดูหมวดปัจจุบัน
/folder off   ปิดหมวดปัจจุบัน
/help         ดูคำสั่งทั้งหมด

หมวดมีผลแยกตามห้องแชตหรือกลุ่ม LINE ถ้าไม่ใช้คำสั่ง /folder ไฟล์จะถูกเก็บในโฟลเดอร์ปกติตามประเภทไฟล์ ถ้าตั้งหมวดที่ยังไม่มีโฟลเดอร์อยู่ ระบบจะสร้างโฟลเดอร์ให้อัตโนมัติ เมื่อใช้งานหมวดเสร็จ ให้พิมพ์ /folder off

สถานะหมวดถูกเก็บใน memory ของ app ถ้า Hostinger redeploy หรือ restart app หมวดปัจจุบันจะถูกล้าง ให้พิมพ์ /folder ชื่อหมวด ใหม่อีกครั้ง

ข้อความตอบกลับหลังบันทึกสำเร็จ

เมื่ออัปโหลดไฟล์เข้า Google Drive สำเร็จ บอตจะรอช่วงสั้น ๆ แล้วสรุปผลเป็นข้อความเดียว:

ทำการบันทึกไฟล์เรียบร้อย 10 ไฟล์

สรุป:
Images/งานประชุม: 6 ไฟล์
Videos/งานประชุม: 2 ไฟล์
Documents/งานประชุม: 2 ไฟล์

โฟลเดอร์:
Images/งานประชุม: https://drive.google.com/drive/folders/...
Videos/งานประชุม: https://drive.google.com/drive/folders/...
Documents/งานประชุม: https://drive.google.com/drive/folders/...

ค่าเริ่มต้นจะรอ 45 วินาทีก่อนส่งสรุป ปรับได้ด้วย environment variable SUMMARY_DELAY_SECONDS ถ้าบันทึกไฟล์ไม่สำเร็จ บอตจะแจ้ง error ในกลุ่ม

ถ้าต้องการให้สมาชิกในกลุ่มเปิดลิงก์ได้ ต้องแชร์โฟลเดอร์ Google Drive ให้สมาชิกเหล่านั้น หรือปรับสิทธิ์เป็น anyone with the link ตามนโยบายความปลอดภัยของคุณ

9. คำสั่งนัดหมายในกลุ่ม LINE

สร้างนัดหมาย:

/นัด ประชุมโครงการ | 2026-07-05 14:00 | ห้องประชุม A

เมื่อบันทึกสำเร็จ บอตจะตอบกลับพร้อมรหัสนัดหมาย:

บันทึกนัดหมายแล้ว

รหัส: A-1234
หัวข้อ: ประชุมโครงการ
เวลา: 5 ก.ค. 2026 14:00
รายละเอียด: ห้องประชุม A

จะแจ้งเตือนเช้าวันนัดหมาย 08:00 และก่อนเวลานัด 10 นาที

ดูนัดหมายที่กำลังจะมาถึง:

/นัดหมาย

ยกเลิกนัดหมาย:

/ยกเลิกนัด A-1234

ดูตัวอย่างคำสั่ง:

/นัด help

ระบบจะตรวจนัดหมายทุก 60 วินาที และส่งแจ้งเตือนเข้า LINE กลุ่มตามเวลาที่ตั้งไว้

ทดสอบระบบนัดหมายหลัง deploy

  1. พิมพ์ /นัด help ในกลุ่ม LINE เพื่อดูตัวอย่างคำสั่ง
  2. สร้างนัดหมายทดสอบที่อยู่ในอนาคต เช่น อีก 15 นาที
  3. พิมพ์ /นัดหมาย เพื่อตรวจว่านัดหมายถูกบันทึกใน MySQL แล้ว
  4. รอให้บอตแจ้งเตือนก่อนเวลานัดหมาย 10 นาที
  5. สร้างนัดหมายอีกอัน แล้วทดสอบยกเลิกด้วย /ยกเลิกนัด รหัส
เวลาที่พิมพ์ในคำสั่งใช้รูปแบบ YYYY-MM-DD HH:mm และอ้างอิงเวลาไทย Asia/Bangkok เช่น 2026-07-05 14:00
การแจ้งเตือนนัดหมายใช้ LINE push message จึงนับรวมกับ quota/แพ็กเกจข้อความของ LINE Official Account

11. การตรวจปัญหา

เปิด URL หลักแล้วไม่ขึ้น JSON

  • ตรวจว่า app deploy สำเร็จ
  • ตรวจ start command ว่าเป็น npm start
  • ตรวจ runtime ว่าเป็น Node.js 20 ขึ้นไป
  • ดู log ใน Hostinger

LINE Verify ไม่ผ่าน

  • ตรวจว่า URL เป็น HTTPS
  • ตรวจว่า URL ลงท้ายด้วย /webhook
  • ตรวจว่า app กำลัง Running
  • ดู log ว่ามี error เรื่อง environment variables หรือไม่

ขึ้น Invalid LINE signature

  • ตรวจค่า LINE_CHANNEL_SECRET
  • ต้องใช้ secret จาก channel เดียวกับ webhook ที่ตั้งไว้
  • หลังแก้ env แล้ว redeploy หรือ restart app

อัปโหลดเข้า Google Drive ไม่ได้

  • ตรวจว่าเปิด Google Drive API แล้ว
  • ตรวจว่าแชร์โฟลเดอร์หลักให้ service account เป็น Editor แล้ว
  • ตรวจค่า GOOGLE_DRIVE_ROOT_FOLDER_ID
  • ตรวจว่า GOOGLE_SERVICE_ACCOUNT_BASE64 คัดลอกครบ ไม่มีเว้นวรรคหรือขึ้นบรรทัดผิด

ส่งไฟล์แล้วไม่เห็นไฟล์ใน Drive

  • ตรวจ log ใน Hostinger
  • ตรวจว่าบอตยังอยู่ในกลุ่ม LINE
  • ตรวจว่าไฟล์ที่ส่งเป็น message ประเภท image, video หรือ file
  • ลองส่งไฟล์ขนาดเล็กก่อน เช่น รูปภาพ 1 รูป

คำสั่งนัดหมายบอกว่ายังไม่ได้เปิดใช้งาน

  • ตรวจว่าใส่ DB_HOST, DB_USER, DB_PASSWORD และ DB_NAME ครบแล้ว
  • ตรวจว่า MySQL user มีสิทธิ์สร้างตารางและเขียนข้อมูลใน database
  • หลังแก้ Environment Variables ให้ restart หรือ redeploy app
  • ดู log ว่ามีข้อความ Appointment database is ready หรือ error จาก MySQL หรือไม่

บอตไม่แจ้งเตือนนัดหมาย

  • ตรวจว่า app ยัง Running อยู่ใน Hostinger
  • ตรวจค่า REMINDER_MORNING_HOUR และ REMINDER_CHECK_INTERVAL_SECONDS
  • ตรวจว่าเวลานัดหมายยังไม่ผ่านไปแล้ว
  • ตรวจ LINE quota เพราะการแจ้งเตือนใช้ push message
  • ดู log ใน Hostinger ว่ามี error ตอนส่ง push message หรือไม่

12. Checklist ก่อนใช้งานจริง

  • Hostinger app เปิด URL หลักแล้วได้ {"ok":true}
  • ตั้ง environment variables ครบ
  • เปิด HTTPS แล้ว
  • LINE webhook URL เป็น https://โดเมนของคุณ/webhook
  • LINE Verify ผ่าน
  • บอตอยู่ในกลุ่ม LINE
  • Google Drive แชร์ให้ service account เป็น Editor
  • ทดสอบส่งรูป วิดีโอ และเอกสารแล้วเข้า Drive ถูกโฟลเดอร์
  • ถ้าใช้ระบบนัดหมาย ต้องเห็น log Appointment database is ready
  • ทดสอบ /นัด, /นัดหมาย และ /ยกเลิกนัด แล้ว
